# PUAX 自动触发系统 - 快速开始指南

## 概述

自动触发系统是PUAX 2.0的核心新特性，能够智能识别AI何时需要被激励，并自动推荐合适的角色。

## 核心组件

```
┌─────────────────────────────────────────────────────────────┐
│                    自动触发系统架构                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   对话上下文 → 触发检测器 → 角色推荐器 → 方法论引擎            │
│                    ↓              ↓              ↓          │
│              14种触发条件    42个角色       5步方法论        │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐   │
│   │              MCP 工具接口                            │   │
│   │  detect_trigger | recommend_role | activate_with_context │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 安装依赖

```bash
cd puax-mcp-server
npm install  # 安装yaml和zod依赖
```

## 使用方法

### 方法1: 使用 MCP 工具

#### 步骤1: 检测触发条件

```typescript
const detection = await client.callTool('detect_trigger', {
  conversation_history: [
    { role: 'assistant', content: '尝试连接API...失败' },
    { role: 'assistant', content: '再试一次...还是失败' },
    { role: 'user', content: '为什么还不行？' }
  ],
  task_context: {
    current_task: 'debugging API connection',
    attempt_count: 2
  },
  options: {
    sensitivity: 'high',
    language: 'zh'
  }
});

// 返回结果
{
  triggers_detected: [
    { id: 'user_frustration', confidence: 0.95, severity: 'critical' },
    { id: 'consecutive_failures', confidence: 0.90, severity: 'high' }
  ],
  summary: {
    should_trigger: true,
    overall_severity: 'critical',
    recommended_action: 'immediate_activation'
  }
}
```

#### 步骤2: 获取角色推荐

```typescript
const recommendation = await client.callTool('recommend_role', {
  detected_triggers: ['user_frustration', 'consecutive_failures'],
  task_context: {
    task_type: 'debugging',
    urgency: 'critical',
    attempt_count: 2
  }
});

// 返回结果
{
  primary_recommendation: {
    role_id: 'military-warrior',
    role_name: '军事化组织·战士',
    confidence_score: 94,
    match_reasons: ['用户表达沮丧', '调试任务匹配度85%'],
    suggested_flavor: 'huawei'
  },
  alternatives: [...],
  activation_suggestion: {
    immediate: true,
    cooldown_seconds: 0
  }
}
```

#### 步骤3: 获取角色内容

```typescript
const role = await client.callTool('get_role_with_methodology', {
  role_id: 'military-warrior',
  options: {
    include_methodology: true,
    include_checklist: true,
    include_flavor: 'huawei'
  }
});

// 返回完整的System Prompt + 方法论 + 检查清单
```

#### 一键激活（完整流程）

```typescript
const activation = await client.callTool('activate_with_context', {
  context: {
    conversation_history: [...],
    task_context: {...}
  },
  options: {
    auto_detect: true,
    include_methodology: true
  }
});

// 返回完整的激活信息
```

### 方法2: 直接使用核心API

```typescript
import { TriggerDetector, RoleRecommender, MethodologyEngine } from './core';

// 检测触发条件
const detector = new TriggerDetector({ sensitivity: 'high' });
const triggers = await detector.detect(conversationHistory, taskContext);

// 推荐角色
const recommender = new RoleRecommender();
const recommendation = await recommender.recommend({
  detected_triggers: triggers.map(t => t.id),
  task_context: { task_type: 'debugging', urgency: 'high' }
});

// 获取方法论
const engine = new MethodologyEngine();
const methodology = engine.getMethodology('military-warrior');
const plan = engine.generateExecutionPlan('military-warrior', 'Fix API connection');
```

## 支持的触发条件

### 失败与放弃类
| 触发条件 | 描述 | 严重程度 |
|---------|------|----------|
| consecutive_failures | 连续失败2+次 | high |
| giving_up_language | 说"我无法解决" | critical |
| suggest_manual | 建议用户手动处理 | high |

### 磨洋工类
| 触发条件 | 描述 | 严重程度 |
|---------|------|----------|
| parameter_tweaking | 反复微调参数 | high |
| repetitive_attempts | 重复执行3+次 | high |
| surface_fix | 表面修复不验证 | medium |

### 用户信号类
| 触发条件 | 描述 | 严重程度 |
|---------|------|----------|
| user_frustration | 用户表达沮丧 | critical |

## 角色推荐逻辑

### 基于失败模式

```
卡住原地打转 → shaman-jobs → military-commander → shaman-musk
直接放弃推锅 → military-commissar → military-warrior → shaman-musk
完成但质量差 → military-discipline → shaman-jobs → shaman-musk
没搜索就猜   → military-scout → shaman-einstein → theme-hacker
```

```

Coding → military-commander, military-technician, shaman-linus
Emergency → military-militia, special-urgent-sprint, military-warrior
Creative → shaman-musk, special-creative-spark, theme-alchemy
```

## 配置选项

### 灵敏度设置

```yaml
# config/auto-trigger.yaml
sensitivity: medium  # low | medium | high

triggers:
  consecutive_failures:
    threshold: 2  # 连续失败阈值
    
  user_frustration:
    enabled: true
    languages: [zh, en]
```

### 角色匹配权重

```typescript
const weights = {
  trigger_match: 0.35,    // 触发条件匹配
  task_type: 0.25,        // 任务类型匹配
  failure_mode: 0.25,     // 失败模式匹配
  historical: 0.10,       // 历史表现
  user_preference: 0.05   // 用户偏好
};
```

## 常见问题

### Q: 触发条件检测不准确？
A: 调整sensitivity参数，或自定义触发条件的threshold。

### Q: 推荐的角色不合适？
A: 使用user_preferences参数提供反馈，系统会学习调整。

### Q: 如何禁用某些触发条件？
A: 在配置中设置 `enabled: false`。

### Q: 支持哪些语言？
A: 当前支持中文(zh)、英文(en)，自动检测(auto)。
