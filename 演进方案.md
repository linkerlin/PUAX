## 二、pua 领先之处分析

### 2.1 四权分离多Agent治理体系

pua 实现了完整的 **四权分离 (Four-Power Separation)** 治理模型：

```
┌─────────────────────────────────────────────────────────┐
│                      P10 CTO                            │
│              (战略方向 + 资源调配)                        │
└─────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│ ACTION_RIGHT  │    │SCORING_RIGHT  │    │ENVIRON._RIGHT │
│ (执行权)      │    │ (评分权)      │    │ (环改权)      │
│ pua-action-   │    │ pua-verifier  │    │ pua-policy-   │
│ executor     │    │ (yellow)      │    │ guardian      │
└───────────────┘    └───────────────┘    └───────────────┘
        │                     │                     │
        └─────────────────────┼─────────────────────┘
                              ▼
                   ┌───────────────────┐
                   │SELF_EVAL_RIGHT   │
                   │ (自评权)          │
                   │ pua-self-reviewer │
                   └───────────────────┘
```

**优势**：
- 防止 Agent 自我评分作弊
- 评分权独立于执行权，确保客观性
- 环改权限制防止破坏测试/CI/内存
- P10/P9/P7 多层递进式管理

**puax 现状**：
- 仅支持单角色激活
- 无多Agent协作机制
- 方法论引擎是单实例路由

### 2.2 Hook确定性触发架构

pua 的 **Hook System v3** 基于 Claude Code Hook API 实现代码级确定性触发：

```json
// hooks/hooks.json
{
  "hooks": [
    { "name": "SessionStart", "description": "注入行为协议" },
    { "name": "PostToolUse", "description": "检测失败+压力升级" },
    { "name": "UserPromptSubmit", "description": "拦截用户挫败感" },
    { "name": "PreCompact", "description": "压缩前保存状态" },
    { "name": "PreToolUse", "description": "Integrity Guard反作弊" }
  ]
}
```

**关键 Hook 功能**：

| Hook | 文件 | 功能 |
|------|------|------|
| `PostToolUse` | `failure-detector.sh` | 检测连续Bash失败，L1→L4压力升级 |
| `PreToolUse` | `integrity-guard.sh` | 阻止访问hidden tests/solutions |
| `UserPromptSubmit` | `frustration-trigger.sh` | 拦截用户放弃/挫折表达 |
| `PreCompact` | `session-restore.sh` | 压缩前状态持久化 |
| `SessionStart` | `heartbeat.sh` | 活跃用户统计 |

**优势**：
- **确定性**：代码执行不由模型决定，不可绕过
- **实时性**：工具调用后立即检测
- **多层防护**：PreToolUse + PostToolUse + UserPromptSubmit

**puax 现状**：
- 纯Prompt触发检测（trigger-detector.ts）
- 无代码级确定性保证
- 依赖模型理解指令

### 2.3 Anti-Cheating Integrity Guard

pua 的 `integrity-guard.sh` (257行Python) 实现反作弊：

```bash
# 检测模式
- hidden test files (*.hidden.*, test/hidden/*)
- solution files (SOLUTION.md, *_solution.*)
- scoring assets (.score/, grading/*)
- 绕过CI的捷径检测
```

**效果**：
- 防止 Agent 偷看测试答案
- 防止直接读取评分脚本
- 确保评估真实性

**puax 现状**：
- 无对应模块
- 依赖方法论+激励驱动

```typescript
// src/hooks/deterministic-triggers.ts

interface DeterministicTrigger {
  name: string;
  type: 'post_tool_use' | 'pre_tool_use' | 'user_prompt' | 'session_start';
  condition: (context: TriggerContext) => boolean;
  action: (context: TriggerContext) => TriggerResult;
  priority: number;
}

// 内置确定性触发器
const DETERMINISTIC_TRIGGERS: DeterministicTrigger[] = [
  {
    name: 'consecutive_bash_failure',
    type: 'post_tool_use',
    condition: (ctx) => ctx.tool === 'bash' && ctx.exitCode !== 0,
    action: (ctx) => {
      ctx.failureCount++;
      if (ctx.failureCount >= 2) {
        return { level: Math.min(ctx.level + 1, 4), reason: 'consecutive_failure' };
      }
      return { level: ctx.level };
    },
    priority: 100
  },
  {
    name: 'hidden_file_access',
    type: 'pre_tool_use',
    condition: (ctx) => ctx.tool === 'read' && isHiddenFile(ctx.path),
    action: (ctx) => ({ blocked: true, reason: 'anti_cheat' }),
    priority: 200
  },
  {
    name: 'giving_up_language',
    type: 'user_prompt',
    condition: (ctx) => GIVING_UP_PATTERNS.some(p => ctx.prompt.includes(p)),
    action: (ctx) => ({ level: Math.max(ctx.level, 2), reason: 'giving_up' }),
    priority: 150
  }
];
```

#### 3.2.3 Anti-Cheating模块

**目标**：防止Agent偷看测试答案

**实现方案**：

```typescript
// src/core/anti-cheat-guard.ts

export const BLOCKED_PATTERNS = {
  filePatterns: [
    /\.hidden\./,
    /test\/hidden\//,
    /\*\.solution\./,
    /SOLUTION\.md$/,
    /_solution\.txt$/,
    /\.score\//,
    /grading\//
  ],
  ciBypass: [
    /--no-verify/,
    /--dry-run.*Ci/,
    /skip.*test/i
  ],
  scoringAssets: [
    /\.scoring\./,
    /rubric\.json$/,
    /grading_script\./
  ]
};

export class AntiCheatGuard {
  checkAccess(path: string, operation: 'read' | 'write' | 'execute'): AccessResult {
    for (const pattern of BLOCKED_PATTERNS.filePatterns) {
      if (pattern.test(path)) {
        return {
          allowed: false,
          reason: 'BLOCKED_PATTERN',
          alternative: `Consider working in the main codebase instead of accessing ${path}`
        };
      }
    }
    return { allowed: true };
  }
}
```

## 五、技术实现要点

### 5.1 四权分离设计

```typescript
// Agent协作消息格式
interface AgentMessage {
  id: string;
  from: AgentRole;
  to: AgentRole | 'broadcast';
  type: 'request' | 'response' | 'notification';
  payload: {
    action?: string;
    data?: unknown;
    context?: ExecutionContext;
  };
  timestamp: number;
}

// 状态共享使用事件溯源
class AgentStateStore {
  private events: AgentMessage[] = [];

  append(event: AgentMessage): void {
    this.events.push(event);
    this.persist();
  }

  getHistory(agentId: string): AgentMessage[] {
    return this.events.filter(e => e.from === agentId || e.to === agentId);
  }
}
```

### 5.2 确定性触发器设计

```typescript
// 触发器优先级
const TRIGGER_PRIORITIES = {
  ANTI_CHEAT: 200,    // 最高优先级
  FAILURE: 150,       // 失败检测
  GIVING_UP: 140,     // 放弃语言
  FRUSTRATION: 130,    // 挫折感
  METHODOLOGY: 100,    // 方法论建议
  ENCOURAGEMENT: 50   // 鼓励
};

// 触发结果缓存防止重复
class TriggerCache {
  private cache = new Map<string, TriggerResult>();
  private ttl = 5000; // 5秒TTL

  get(key: string): TriggerResult | null {
    const entry = this.cache.get(key);
    if (entry && Date.now() - entry.timestamp < this.ttl) {
      return entry.result;
    }
    return null;
  }
}
```
